## PARTCOPYとは？

PARTCOPYは、参考サイトのURLを入力するだけで、そのサイトのデザインパターン（ヒーロー、料金表、FAQ、CTAなど）を**自動で分類・保存**し、それらを**ドラッグ＆ドロップで組み合わせて新しいページを作れる**ツールです。

### 何ができる？

```
参考サイトURL → 自動分析 → パーツ分類 → ライブラリに保存
                                              ↓
               HTML出力 ← プレビュー ← ドラッグ＆ドロップで組み立て
```

- 複数サイトのデザインパターンを一括収集
- AIが18種類のセクションタイプに自動分類
- 視覚的にドラッグ＆ドロップでページを構成
- テキスト・画像をWordPress風にインライン編集
- Monaco（VS Code相当）でHTMLコード直接編集
- AIでデザインの統一感を自動調整
- 完成ページをHTMLファイルで書き出し

### 技術スタック

| 領域 | 技術 |
|------|------|
| フロントエンド | React + Vite + Monaco Editor |
| バックエンド | Express + Puppeteer（ヘッドレスブラウザ） |
| AI分類 | Claude Haiku（セクション分類） |
| AIデザイン統一 | Claude Sonnet 4（HTML生成） |
| データベース | Supabase（PostgreSQL + Storage） |
| ローカルモード | Supabase不要で動作（JSONファイル保存） |

## 画面構成

### ナビゲーション（上部タブ）

```
┌─────────────────────────────────────────────────────────┐
│ PARTCOPY   ビルダー | ライブラリ | プレビュー | プロジェクト   4パーツ 3Canvas 2サイト │
└─────────────────────────────────────────────────────────┘
```

| タブ | 説明 |
|------|------|
| **ビルダー** | メイン画面。左にライブラリ、右にキャンバス。ドラッグ＆ドロップで構築 |
| **ライブラリ** | 全セクションの一覧。フィルタ・検索・ソート |
| **プレビュー** | キャンバスの完成イメージ。HTML出力ボタンあり |
| **プロジェクト** | キャンバスの状態を保存・復元 |

右上の数値：
- **パーツ** — ライブラリに保存されたセクション数
- **Canvas** — 現在キャンバスに乗っているブロック数
- **サイト** — 分析済みのユニークドメイン数

### URL入力バー

```
┌──────────────────────────────────────────────┐
│  [自社サイト分析] [参考パターン収集]          │
│  🔍 参考にしたいサイトのURLを入力   [分析する] │
└──────────────────────────────────────────────┘
```

- **自社サイト分析** — 自社サイトを分析するモード
- **参考パターン収集** — 参考サイトのパターンを収集するモード
- URLを入力して「分析する」を押すと分析開始

## 使い方：基本フロー

### Step 1: サイトを分析する

1. URL入力バーに参考サイトのURLを入力
2. 「**分析する**」をクリック
3. ターミナル風の進捗UIが表示される：

```
$ partcopy analyze https://example.com
  fetch --headless        ページ取得中...
  render --javascript     レンダリング中...
  detect --sections       セクション検出中... (34パーツ検出)
  classify --ai haiku     AI分類中...
  store --deduplicate     保存中...
  done in 45s
```

4. 完了するとライブラリにセクションが追加される

**所要時間の目安：**
- シンプルなサイト（10〜20セクション）: 30秒〜1分
- 大規模サイト（50〜100セクション）: 2〜5分
- 画像・フォントが多いサイト: 5〜10分

### Step 2: ライブラリからパーツを選ぶ

ライブラリには分析結果が18種類に分類されて表示されます。

**フィルタ方法：**
- 左サイドバーの**業種カテゴリ**で絞り込み
- 左サイドバーの**セクションの種類**で絞り込み（ヒーロー、FAQ、料金など）
- 上部の**検索バー**でテキスト検索
- **IMG / CTA / FORM** チップで画像あり・CTAあり・フォームありを絞り込み
- **ソート**で新しい順・スコア順・種類順・参照元順を切替

### Step 3: ドラッグ＆ドロップでページを組む

1. ライブラリのカードを**ドラッグ**してキャンバスにドロップ
2. または「**+ 追加**」ボタンをクリック
3. ブロック間に表示される「**+**」ゾーンに挿入も可能
4. **ドラッグハンドル（☰）** でブロックの順番を変更
5. **↑↓ボタン**でも移動可能

### Step 4: 編集する

**方法A：ビジュアル編集（WordPress風）**
1. ブロックの「**編集**」ボタンをクリック
2. プレビュー内の**テキストをクリック** → その場で編集
3. **画像をクリック** → 新しい画像URLを入力して差替え
4. 右パネルの**ノードインスペクター**で詳細編集

**方法B：コード編集（Monaco）**
1. ブロックの「**</>**」ボタンをクリック
2. 左パネルにHTMLコードが表示される
3. 直接編集 → **Cmd+S**で保存
4. 右パネルにリアルタイムプレビュー

### Step 5: プレビュー＆書き出し

1. **プレビュー**タブに切替
2. 全ブロックが縦に並んだ完成イメージを確認
3. 「**HTML出力**」をクリック → `partcopy-export.html` がダウンロード

**オプション：AI デザイン統一**
- ブランドカラー・業種・ターゲット層を設定
- AIが全セクションのデザインを統一して再生成
- 「**AI統一プレビュー**」で確認後にエクスポート

### Step 6: プロジェクトに保存

1. **プロジェクト**タブに切替
2. プロジェクト名を入力して「**作成**」
3. 「**Canvasを保存**」で現在の状態を保存
4. 次回「**読み込む**」で復元

## 各機能の詳細

### セクション分類（18種類）

AIが自動で以下のカテゴリに分類します：

**ページ上部**
| カテゴリ | 説明 | 色 |
|---------|------|-----|
| メニュー | ヘッダーナビゲーション | 紫 |
| メインビジュアル | ファーストビューのヒーロー画像 | 青 |

**メインコンテンツ**
| カテゴリ | 説明 | 色 |
|---------|------|-----|
| 特徴・サービス紹介 | サービスの特長をカードやグリッドで | 緑 |
| お客様の声 | ユーザーレビュー・導入事例 | ピンク |
| 数字で見る実績 | 実績数値・統計データ | 黄緑 |
| 導入企業ロゴ | クライアント・パートナーロゴ | 紫 |
| 料金プラン | 価格テーブル・比較表 | 紫 |
| よくある質問 | Q&Aアコーディオン | ティール |
| 読み物・説明 | テキスト主体のコンテンツ | グレー |
| お知らせ一覧 | ニュース・ブログ | 紫 |
| 写真ギャラリー | 画像ギャラリー | シアン |
| 会社情報 | 会社概要・チーム紹介 | 緑 |
| 沿革・ステップ | タイムライン・プロセス | 水色 |
| 採用情報 | 求人・キャリア | シアン |
| カード | 単体の情報カード | オレンジ |

**コンバージョン**
| カテゴリ | 説明 | 色 |
|---------|------|-----|
| アクションボタン | CTA・申し込みボタン | オレンジ |
| 問い合わせフォーム | コンタクトフォーム | オレンジ |

**ページ下部**
| カテゴリ | 説明 | 色 |
|---------|------|-----|
| フッター | フッター | グレー |

> メニュー・フッターはキャンバスには追加されません（自動除外）。

### 画像URL表示機能

ライブラリの各カードで**IMG**ボタンが表示されるセクション（画像含むもの）があります。

1. **IMGボタン**をクリック → セクション内の画像URL一覧が展開
2. URLをクリック → 新しいタブで画像を表示
3. **copy**ボタン → クリップボードにURLをコピー

画像をダウンロードしたい場合は、URLを新しいタブで開いて右クリック保存。

### コードパネル

ライブラリの各カードの「**</>**」ボタンから開けます。

- セクションのHTMLソースコードを表示
- 「**コードをコピー**」でクリップボードにコピー
- 「**プロンプト付きコピー**」でAIに渡せる構造化プロンプト付きでコピー
  - セクション種類・バリアント名
  - 含まれる要素（画像/CTA/フォーム）
  - 参照元ドメイン
  - 実装ガイダンス

### AI デザイン統一

プレビュー画面で使える機能。複数サイトから集めたバラバラなデザインを統一します。

**設定項目：**
- **ブランドカラー**（カラーピッカー）
- **業種**（SaaS, EC, コーポレート, 医療, 教育, 飲食, 不動産, 金融, メディア, その他）
- **ターゲット層**（自由入力。例：「30代ビジネスパーソン」）

**処理内容：**
1. 各セクションのレイアウト構造（見出し数・ボタン数・カード数など）を抽出
2. Claude Sonnet 4がゼロから新しいHTML/CSSを生成
3. 統一されたデザインでプレビュー表示
4. HTMLファイルとしてエクスポート可能

> **コスト：** 1回あたり約$0.06〜0.10（約10〜15円）

## セキュリティ

### 実装済みの対策

| カテゴリ | 対策 |
|---------|------|
| **認証** | APIキー認証（X-API-Keyヘッダー）。全書き込みエンドポイントに適用 |
| **レート制限** | グローバル: 3000回/15分、クロール: 10回/時間、AI: 20回/時間 |
| **SSRF防止** | localhost・プライベートIP・内部ホスト名への接続をブロック |
| **入力検証** | 全IDのUUID形式チェック、文字列長制限、HTMLタグ除去 |
| **パストラバーサル防止** | ストレージパスの`../`や絶対パスをブロック |
| **HTTPヘッダー** | Helmet（CSP, X-Frame-Options, HSTS等） |
| **CORS** | 許可オリジンのホワイトリスト制。不許可はエラー |
| **リクエストサイズ** | JSONボディは1MBまで |

### 環境変数（.env）

```env
# サーバーポート
PARTCOPY_API_PORT=3002

# APIキー認証（サーバーとクライアントで同じ値を設定）
PARTCOPY_API_KEY=your-secret-key
VITE_PARTCOPY_API_KEY=your-secret-key

# Supabase（ローカルモードでは不要）
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# Anthropic API（AI分類に必要）
ANTHROPIC_API_KEY=sk-ant-xxxxx
```

## トラブルシューティング

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「Too many requests」 | レート制限に到達 | 15分待つか、server/index.tsのmax値を調整 |
| 分析が終わらない | 重いサイト / 大量のアセット | 5〜10分待つ。タイムアウト後は自動で完了扱い |
| セクションが分類されない | ANTHROPIC_API_KEY未設定 | .envにキーを追加 → ヒューリスティック分類にフォールバック中 |
| CSSが効かない | Helmetの CSPヘッダー | render/editable-renderではCSP自動解除済み |
| ポートが使用中 | 前のプロセスが残っている | `kill -9 $(lsof -ti :3002 -ti :5180)` |
| Supabase接続エラー | キーが未設定 or 期限切れ | ローカルモードに自動フォールバック |
| 「Invalid API key」 | クライアントとサーバーのキー不一致 | .envのPARTCOPY_API_KEYとVITE_PARTCOPY_API_KEYを同じ値に |

## 起動方法

```bash
# 依存関係インストール
npm install

# 開発サーバー起動（API + Worker + Vite を同時起動）
npm run dev

# ブラウザで開く
open http://127.0.0.1:5180
```

3つのプロセスが同時起動します：
- **dev:server** — APIサーバー（Express, ポート3002）
- **dev:worker** — バックグラウンドワーカー（クロール処理）
- **dev:client** — フロントエンド（Vite, ポート5180）
